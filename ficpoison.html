Ao3 AI Poisoner

Sick and tired of hearing about AI scraping Ao3? Want to try and fight back? Try this.


How does it work?

Just copy your HTML-formatted fic text from Ao3, paste it into the box on the left, and hit the "Apply poison" button to insert random junk text that will hopefully confuse any AI scrapers. Copy the poisoned text, and replace your existing HTML-formatted fic text. Then, be sure to either create and apply a new work skin with the following CSS code, or add it to the workskin currently on your fic. This is what will allow the poison to be hidden from your readers, while still being visible to an AI scraper.

#workskin .poison { display: none; }

Always check your fic with the preview button before posting! I also recommend keeping an unpoisoned backup for ease of editing later on, if necessary.

Is your fic not in HTML format? Just paste it into the rich text editor when creating or editing a work or chapter on Ao3, then switch to the HTML editor. Ao3 should automatically format it into HTML.


How will this affect my readers?

So long as they keep Creator's Style on, it should be completely unnoticeable, even to a screen reader. However, if they try to turn Creator's Style off or download the fic through Ao3's built-in download buttons, the poison text will become visible.

If you want to see examples, you can check out my Ao3 account. This formatting is on most of my fics, and I always add a heads-up to the author's notes at the beginning when I use it.


Will this website record my data?

No, this site will not save or record your data in any way. The full code for this website is available for you to view at any time on my GitHub page at https://github.com/TricksOfLoki/TricksOfLoki.github.io.


Is there an offline version?

Yes, you can download a local version at https://github.com/TricksOfLoki/fic-poison. This version also includes the option to change the name of the CSS class used, and an alternate style where poison is inserted in large chunks between paragraphs rather than at random.
